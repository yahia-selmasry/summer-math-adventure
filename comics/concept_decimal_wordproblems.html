✅ Example 2 — Multiplying Decimals
"Juice boxes cost $1.35 each. Max buys 8 of them. What is the total cost?"
1 Price: $1.35; Quantity: 8
3 Operation: MULTIPLY
4 Ignore decimal: 135 × 8 = 1,080
   Count 2 decimal places → $10.80
5 Estimate: $1 × 8 = $8, close ✓
   (real is a bit more since 1.35 > 1)

✅ Example 4 — Comparing Decimals (Measurement)
"Three swimmers' times: Zara: 1.47 min, Max: 1.39 min, Leo: 1.4 min. Order from fastest to slowest."
Trick Pad with zeros so all have same decimal places:
Zara: 1.47
Max: 1.39
Leo: 1.40
Compare hundredths: 39 < 40 < 47
Fastest → Slowest:
Max (1.39) · Leo (1.40) · Zara (1.47)
Lower time = faster swimmer!
⚠️ CLASSIC MISTAKES
❌ NOT lining up decimal points before adding/subtracting
❌ Thinking 1.9 < 1.39 (it doesn't have more digits so it must be smaller — WRONG!)
❌ Forgetting to count decimal places when multiplying
❌ Writing $7.5 instead of $7.50 (always 2 places for money)
❌ Dropping the decimal point in the final answer
❌ Not including units ($, m, kg, etc.)
💡 PRO TIPS
✓ LINE UP decimal points, then add zeros to fill empty spots
✓ Compare decimals: pad with zeros → compare like whole numbers
✓ Multiply: ignore decimal → multiply → count places
✓ Money always has 2 decimal places: $4.70 not $4.7
✓ Estimate first: round to nearest dollar or whole number
✓ Check subtraction by adding: sum = original total
